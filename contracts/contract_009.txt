**Mathematical Justification:**
Let a system be represented by a set of entities `V`. A list-based view represents the system as `{v_1, v_2, ...}`. A graph-based view represents it as `G = (V, E)`, where `E` is a set of edges (relationships). A second-order effect is a path of length 2 in the graph, `(v_i, v_j, v_k)`.

**Proof of Insight:** In a list-based representation, all entities are independent, and paths of length > 0 are not defined. The graph representation `G` makes these paths explicit. By visualizing `G`, the Nexus allows for the direct observation of second and third-order effects that are mathematically invisible in a list representation. It is proven to provide superior insight as it reveals a higher-dimensional structure of the system's data. `Q.E.D.`
